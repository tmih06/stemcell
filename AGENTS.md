# StemCell — Agent Instructions

## Project Wiki

- Before making code changes, read `wiki/index.md`, `wiki/coverage-manifest.md`, and the wiki pages relevant to the files, entrypoints, flows, contracts, tests, or config being changed.
- The wiki is a source locator and relationship map. Use it to find the right source, then verify behavior in code before editing.
- When code changes alter durable behavior, architecture, commands, configuration, tests, public APIs, file responsibilities, contracts, or workflows, update the affected wiki pages in the same task.
- Update `wiki/coverage-manifest.md` when source areas are added, removed, renamed, excluded, generated, or reorganized.
- Update source maps when files or directories are added, removed, renamed, or given new responsibilities.
- Update entrypoint, flow, contract, or verification pages when runtime behavior, public APIs, schemas, env vars, persistence, integrations, build, test, CI, or deployment behavior changes.
- Do not mark work complete if relevant wiki locator information is stale relative to the code changes.
